# Техническое задание  
## Sputnik Studio v8 (React Edition)

---

## 1. Цель и задачи

**Цель** — создать современную, модульную и расширяемую веб-платформу для проектирования AV-систем, заменив монолитную версию v7 на React-приложение с устранением архитектурных проблем (скрытые контейнеры, конфликты стилей, сложность поддержки).

**Задачи**:
1. Перенести весь функционал v7 (проекты, тракты, калькуляторы, шаблоны) на React с TypeScript.
2. Внедрить роутинг (React Router) для полной изоляции разделов.
3. Использовать Redux Toolkit для единого состояния.
4. Перейти на IndexedDB (Dexie) для локального хранения данных.
5. Разработать компонентную архитектуру, облегчающую дальнейшее расширение (например, добавление графического редактора блок-схем).
6. Обеспечить полную стилистическую совместимость с предыдущей версией (светлая/тёмная темы, стеклянные карточки, адаптивность).

---

## 2. Функциональные требования

### 2.1. Общая структура
- **Роутинг**: четыре основные страницы:
  - `/dashboard` — главная панель управления (виджеты, карусель активных проектов)
  - `/projects` — управление проектами (список, создание, редактирование)
  - `/calculations` — все калькуляторы и работа с трактами (сайдбар + основная область)
  - `/templates` — конструктор документов (шаблоны)
- **Сайдбар** показывается **только** на странице `/calculations`. В нём расположены аккордеоны с настройками (видео, сеть, LED, звук, ВКС, эргономика, тракты, статистика, энергия, управление).

### 2.2. Управление проектами
- CRUD проектов.
- Поля: название, бюджет (руб), инженер, руководитель проекта, дата начала, статус (пресейл → стадия П → стадия Р → монтаж → завершён).
- **Дорожная карта** в двух вариантах:
  - **Плановая** — задаётся пользователем (даты начала/окончания этапов).
  - **Фактическая** — заполняется по мере выполнения (фактические даты выполнения этапов).
- Возможность сравнивать план и факт (визуально, а также формировать отчёт о расхождениях).
- Прогресс может рассчитываться автоматически на основе статуса или задаваться вручную.
- Приоритет (срочность) — визуально выделяется цветом.
- Встречи и закупки: динамические списки с возможностью добавления/удаления.
- При удалении проекта — подтверждение.

### 2.3. Тракты (AV‑тракты)
- Создание нескольких трактов (путей передачи сигнала). У каждого тракта уникальный идентификатор.
- В тракте можно добавлять/удалять устройства из базы моделей (источники, передатчики, матрицы, приёмники, дисплеи, коммутаторы, LED‑процессоры, сплиттеры, переключатели, Dante-устройства).
- Автоматический расчёт:
  - суммарная задержка (сумма латентностей всех устройств)
  - общая мощность (Вт)
  - битрейт (на основе настроек видео и коэффициентов устройств)
  - использование портов коммутаторов (PortManager)
  - потребление PoE (если устройство поддерживает)
- Возможность переключаться между просмотром **активного тракта** (детально) и **списком всех трактов** (обзор).
- Кнопка «Новый тракт» — создаёт новый пустой тракт и делает его активным.
- Редактирование тракта: добавление устройств через модальное окно, выбор модели из выпадающего списка.
- Удаление тракта — с подтверждением.

### 2.4. Графические построения (блок-схемы)
- **Модуль визуализации трактов** (отдельная страница или режим в разделе «Расчёты»).
- Отображение устройств тракта в виде интерактивной блок-схемы (узлы и связи).
- Возможность перетаскивания устройств, масштабирования (zoom/pan), бесконечный холст.
- Экспорт схемы в SVG (векторный формат).
- Синхронизация с данными тракта: добавление/удаление устройства в текстовом редакторе автоматически обновляет схему, и наоборот (двусторонняя привязка).
- В дальнейшем — возможность рисовать произвольные связи между устройствами (создание собственных трактов в графическом режиме).

### 2.5. Калькуляторы (в сайдбаре)
- **Видео**: выбор разрешения, субдискретизации, FPS, цветового пространства, глубины цвета. Расчёт битрейта в реальном времени.
- **Сеть**: выбор среды, multicast, QoS, тип сети, синхронизация, резервирование. Статистика: битрейт, загрузка, использованные порты, статус multicast/QoS.
- **LED**:
  - По кабинетам: шаг пикселя, размеры кабинета, количество кабинетов по горизонтали/вертикали → расчёт итогового разрешения, площади, мощности.
  - По разрешению: целевое разрешение, шаг пикселя → расчёт физических размеров и площади.
- **Звук**:
  - SPL на расстоянии: чувствительность, мощность, расстояние, запас по звуку, усиление помещения.
  - Падение SPL от расстояния: начальное и конечное расстояние, начальный уровень.
  - Изменение SPL от мощности: начальная и конечная мощность, начальный уровень.
  - RT60: объём помещения, площадь, средний коэффициент поглощения.
  - Подбор громкоговорителей: требуемый SPL, чувствительность громкоговорителя, мощность.
- **ВКС**:
  - Расчёт битрейта кодеков: выбор кодека (TrueConf, WebRTC, H.264, H.265), разрешение, FPS, количество участников.
  - Многоточечный вызов: количество участников → расчёт нагрузки на MCU.
- **Эргономика ЭКП**: калькулятор углов обзора, высоты установки, расстояния до экрана.
- **Питание**:
  - Расчёт ИБП: суммарная мощность оборудования, требуемая автономия → выбор ИБП.
  - Расчёт кондиционирования: тепловыделение, запас по охлаждению → требуемая мощность охлаждения.

### 2.6. Шаблоны документов
- Генерация документов (пояснительная записка, спецификация, акт выполненных работ) на основе выбранного шаблона.
- Выбор подсистем (отображение, ВКС, звукоусиление, LED-экраны, управление) для включения в документ.
- Привязка к конкретному проекту — данные проекта подставляются в документ.
- Предварительный просмотр текста документа с возможностью копирования.

### 2.7. Хранение данных
- Все данные (проекты, тракты, настройки калькуляторов) сохраняются в **IndexedDB** (Dexie).
- Поддержка экспорта/импорта JSON (полный бэкап или выборочно).
- При первом запуске — создание демонстрационных проектов (как в v7).

### 2.8. Интерфейс, темы и компоненты
- Светлая и тёмная темы (переключение через кнопку в топбаре).
- Стеклянные карточки, скруглённые углы, плавные переходы.
- Адаптивность под планшеты и мобильные устройства (медиазапросы).
- Кнопки и поля ввода единообразны, с эффектами наведения и фокуса.

### 2.9. Тестирование и самодиагностика
- Автоматические проверки корректности данных (несоответствие битрейта пропускной способности сети, превышение PoE-бюджета).
- Встроенные уведомления об ошибках или потенциальных проблемах (жёлтые/красные предупреждения в интерфейсе).
- Возможность запуска самодиагностики проекта (проверка всех трактов, питания, задержек).
- Юнит-тесты на ключевые функции (расчёты, валидация) с использованием Jest.

### 2.10. Карусель активных проектов
- **Альтернативные библиотеки** (на выбор разработчика с обоснованием):
  - **Embla Carousel** — легковесная, без зависимостей, с высокой производительностью.
  - **Keen-Slider** — современная, гибкая, поддерживает сенсорное управление.
  - **CSS-слайдер** — на основе `scroll-snap` (минималистично, без JS-накладок).
- Требования: адаптивность, возможность скролла пальцем, поддержка кнопок навигации, пагинация.

### 2.11. Иконки
- Вместо Font Awesome 6 предлагается использовать **Phosphor Icons** (https://phosphoricons.com) или **Lucide** (https://lucide.dev).
- Преимущества: более современный дизайн, единая стилистика, легкость кастомизации, поддержка React-компонентов.
- Возможность выбора: на этапе разработки интеграция будет сделана с возможностью замены.

### 2.12. Сайдбар (альтернативные концепции)
- Текущий сайдбар (вертикальный аккордеон) сохраняется как один из вариантов, но предлагаются для рассмотрения альтернативы:
  1. **Выдвижная панель** (по клику на иконку) — экономит место, особенно на узких экранах.
  2. **Контекстная панель инструментов** — отображается только при активном калькуляторе, вместо постоянного присутствия.
  3. **Вкладки (табы) в верхней части области расчётов** — группировка настроек по вкладкам, сайдбар убирается.
- **Выбор** будет сделан после прототипирования и тестирования с целевыми пользователями. Критерии: удобство, скорость доступа к настройкам, отсутствие визуального шума.

## 3. Технические требования

| Компонент | Выбор |
|-----------|-------|
| Язык | TypeScript (строгая типизация) |
| Фреймворк | React 18+ |
| Сборка | Vite |
| Состояние | Redux Toolkit |
| Роутинг | React Router v6 |
| Хранилище | Dexie (IndexedDB) |
| Стили | CSS Modules + глобальные переменные (темы) |
| Drag & Drop (для трактов) | react-dnd (или SortableJS, но позже) |
| Графический редактор (в перспективе) | React Flow (xyflow) |
| Тестирование (в будущем) | Jest + React Testing Library |
| Иконки | Phosphor Icons или Lucide (на выбор) |
| Карусель | Embla / Keen-Slider / CSS scroll-snap (на выбор) |

**Требования к коду**:
- Модульная структура (`features`, `components`, `pages`, `store`, `utils`).
- Каждый слайс Redux изолирован.
- Компоненты — функциональные, с хуками.
- Отсутствие прямых обращений к DOM (всё через React).
- Поддержка локального `storage` для тем, состояния сайдбара, сворачивания аккордеонов.

## 4. Этапы разработки (приоритеты)

1. **Подготовка инфраструктуры** (создание репозитория, настройка Vite, TypeScript, Redux, роутинг).
2. **Перенос базовых страниц** (Dashboard, Projects, Calculations, Templates) с заглушками.
3. **Реализация хранилища IndexedDB** (модели Project, Tract).
4. **Модуль проектов** (полный CRUD, дорожная карта план/факт, виджеты, карусель).
5. **Модуль трактов** (создание трактов, добавление устройств, расчёты, переключение вида).
6. **Калькуляторы** (видео, сеть, LED, звук, ВКС, эргономика, питание) — по одному.
7. **Самодиагностика и уведомления**.
8. **Шаблоны документов** (генерация, привязка к проекту).
9. **Графический редактор блок-схем** (может быть вынесен в отдельный этап после основного функционала).

## 5. Критерии приемки

- Весь функционал v7 воспроизведён и работает без ошибок.
- Переключение между страницами не вызывает утечек памяти и конфликтов DOM.
- Стили не конфликтуют между разделами (карточки проектов не появляются на странице расчётов).
- Сайдбар показывается только на `/calculations`, аккордеоны работают.
- Кнопка «Новый тракт» находится вверху контейнера (проблема v7 решена).
- Графическая схема тракта отображает корректные связи, экспорт в SVG работает.
- Дорожная карта поддерживает план и факт, сравнение отображается.
- Самодиагностика выявляет все типовые ошибки (превышение битрейта, нехватка PoE и т.д.).
- Альтернативная карусель работает без багов на всех устройствах.
- Иконки из выбранного набора выглядят единообразно.
- Экспорт/импорт JSON сохраняет и восстанавливает все данные.
- Проект собирается командой `npm run build` без ошибок.
- Приложение отзывчиво на мобильных устройствах.
